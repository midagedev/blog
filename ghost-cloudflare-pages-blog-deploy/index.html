ghost + cloudflare pages로 블로그 셋팅

cloudflare의 기능을 이용해 한시간만에 도메인+호스팅+배포+어낼리틱스 싹 붙인 충격적인 경험

시작점

chatgpt에서 주고받는 질문 답변들이 정확도가 조금 떨어지더라도 나와 비슷한 궁금증을 가진 분들에게 도움이 될 수 있겠다 싶어서 이 내용을 공유할 블로그를 만들어봐야겠다는 생각이 들었음

무지성 도메인 구입

chatgpt에게 도메인명을 추천받았음, midagedev라는 키워드가 구글에서 검색되지 않는 것을 확인하고 바로 gmail 계정도 하나 파고, 도메인도 구매. 도메인은 예전에 google domains 사업 접는다는 뉴스에 cloudflare에서 츄라이츄라이 했던 것이 기억나서 cloudflare에서 구입, 1년에 9달라 정도

CRM툴 선정

jekyll이나 hexo는 이전에 사용해봤고 뭐 색다른 것이 없을까 싶어 찾아보다가 ghost를 보고 사용해보기로 하였음, 셋업하고 첫 글 쓰는데까지 20분 안걸렸을 정도로 빠르게 사용해볼 수 있도록 준비가 된 툴이었다.

Cloudflare Pages에 배포 셋팅

이 글을 참조하여 진행하였음. ghost에서는 SSG를 직접 지원해주지는 않아서 이를 대신해주는 ghost-static-site-generator를 이용하여 static파일을 만들고 github repository에 push한 후에 cloudflare의 pages와 github연동, 이제 remote에 commit이 생길때마다 자동으로 배포됨, 그리고 도메인 셋팅 이 과정이 10분이 채 안걸렸을 정도로 빠르게 진행되었음

Cloudflare Web Analytics

어낼릭틱스도 뭘 붙일까 하다가 cloudflare껄 붙여봄, 아.. 이건 참, 그냥 도메인 선택하고 클릭클릭 했더니 자동으로 블로그에 추적태그가 추가됨🤯  이런 경험은 처음해보는 것 같다. 도메인부터, 호스팅, 배포, 추적까지 논스톱으로 진행이 되어버렸다. cloudflare진짜 대단한듯

대략  ghost셋팅에 30분, 글쓰는데 30분, cloudflare셋팅하는데 60분 정도 걸린듯 하다.
