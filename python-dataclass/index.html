python의 dataclass

데이터클래스에 대해 알려줘

dataclass는 Python 3.7부터 표준 라이브러리인 dataclasses 모듈에 추가된 데코레이터입니다. 기본적으로 dataclass는 클래스 내의 필드들을 기반으로 초기화, 비교, 출력 메서드 등을 자동으로 생성해 줍니다. dataclass를 사용하면 반복적인 코드를 작성하지 않고도 깔끔하게 데이터를 저장하고 사용하는 클래스를 작성할 수 있습니다.

예제를 통해 이해해봅시다:


from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float
    z: float = 0.0  # 기본값 설정도 가능

위의 예제에서 Point 클래스는 x, y, z 세 개의 필드를 가집니다. dataclass 덕분에 아래와 같은 기능들이 자동으로 제공됩니다:

  • 초기화 메서드 __init__
  • 표현 메서드 __repr__
  • 비교 메서드 __eq__

사용 예:


p1 = Point(1.0, 2.0)
print(p1)  # Point(x=1.0, y=2.0, z=0.0)

p2 = Point(1.0, 2.0, 3.0)
print(p2)  # Point(x=1.0, y=2.0, z=3.0)

print(p1 == p2)  # False

dataclass의 주요 기능 및 옵션:

  1. Field Defaults : 필드의 기본값을 지정할 수 있습니다.
  2. Type Annotations : 필드에 타입 힌트를 제공해야 합니다. 그렇지 않으면 dataclass는 해당 필드를 무시합니다.
  3. Ordering : order=True 옵션을 사용하면, 객체 간의 순서 비교를 위한 메서드들(__lt__, __le__, 등)이 자동으로 생성됩니다.
  4. Immutability : frozen=True 옵션을 사용하면, 불변 객체를 만들 수 있습니다. 이렇게 하면 필드 값을 변경하려고 할 때 예외가 발생합니다.

dataclass는 보일러플레이트 코드 작성을 줄여주므로, 데이터를 다루는 클래스를 작성할 때 매우 유용합니다.

python 3.7 이전에는 그럼 어떻게 대응했을까

dataclass`가 도입되기 전에, 파이썬에서는 여러 가지 방법으로 데이터를 담는 클래스를 정의하고 사용했습니다. 아래는 주요한 접근 방법들입니다:

  1. 일반적인 클래스 정의 : 직접 __init__, __repr__, __eq__ 등의 매직 메서드를 구현했습니다.

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"
    
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
  1. namedtuple : collections 모듈에서 제공하는 namedtuple은 간단한 불변 데이터 구조를 정의하는데 사용되었습니다. 단, namedtuple 인스턴스는 수정할 수 없습니다.

from collections import namedtuple

Point = namedtuple("Point", ["x", "y"])
  1. attrs : dataclass가 도입되기 전에 attrs 라이브러리는 데이터 클래스 생성에 대한 강력한 도구로 인기를 얻었습니다. attrs는 dataclass와 유사한 기능을 제공하지만, 더 많은 커스터마이징과 유틸리티를 제공합니다.

import attr

@attr.s
class Point:
    x = attr.ib()
    y = attr.ib()

이러한 접근 방법들은 여전히 유효하며, 특정 상황에 따라 dataclass보다 더 적합할 수 있습니다. 그러나 dataclass가 도입되면서 표준 라이브러리에서 제공하는 간결하고 강력한 도구로 인기를 얻게 되었습니다.

django에서 dataclass의 활용

Django는 ORM 기반의 웹 프레임워크이기 때문에, dataclass를 사용하는 것이 항상 직관적이지는 않습니다. 그러나 특정 경우에 dataclass를 활용하여 코드의 가독성과 유지보수성을 높일 수 있습니다.

1. Django Form 대체하기

전통적으로 Django에서는 forms.ModelForm 또는 forms.Form을 사용하여 폼 데이터를 처리합니다. 그러나 경우에 따라 dataclass를 사용하여 form 데이터의 구조와 유효성 검사 로직을 분리할 수 있습니다.


from dataclasses import dataclass
from django import forms

@dataclass
class UserRegistrationData:
    username: str
    password: str
    email: str

class UserRegistrationForm(forms.Form):
    username = forms.CharField(max_length=100)
    password = forms.CharField(widget=forms.PasswordInput)
    email = forms.EmailField()

    def clean(self):
        cleaned_data = super().clean()
        return UserRegistrationData(
            username=cleaned_data.get("username"),
            password=cleaned_data.get("password"),
            email=cleaned_data.get("email")
        )

2. QuerySet 데이터 변환하기

Django의 QuerySet에서 반환되는 모델 인스턴스 대신에, 데이터 전송 객체나 특정 용도로 구조화된 객체를 반환하고 싶을 때 dataclass를 사용할 수 있습니다.


from dataclasses import dataclass
from myapp.models import Blog

@dataclass
class BlogSummary:
    title: str
    author_name: str

def get_blog_summaries():
    blogs = Blog.objects.all()
    return [BlogSummary(title=blog.title, author_name=blog.author.name) for blog in blogs]

3. API 응답 구조화하기

API 응답의 형태를 dataclass로 정의하여, 응답의 구조와 데이터를 명확하게 표현할 수 있습니다.


from dataclasses import dataclass
from rest_framework.response import Response

@dataclass
class UserResponse:
    id: int
    name: str
    email: str

def user_detail_view(request, user_id):
    user = get_user_from_database(user_id)  # 여기서는 예시로, 실제로 데이터를 가져오는 로직이 필요합니다.
    response_data = UserResponse(id=user.id, name=user.name, email=user.email)
    return Response(response_data)

4. 비즈니스 로직 내부의 데이터 전달

복잡한 비즈니스 로직을 처리하는 동안 다양한 함수나 메서드에 데이터를 전달할 때 dataclass를 사용하여 데이터의 구조를 명확히 할 수 있습니다.


from dataclasses import dataclass

@dataclass
class OrderCalculationContext:
    base_price: float
    discount: float
    tax: float

def calculate_total_price(context: OrderCalculationContext) -> float:
    return context.base_price - context.discount + context.tax

이러한 방법들은 dataclass를 Django 프로젝트에 통합하는 여러 방법 중 일부입니다. 사용할 때의 환경과 요구 사항에 따라 적절한 방법을 선택하면 됩니다.

별로 장점이 없는것 같아

dataclass의 주요 장점은 반복적인 코드의 제거와 데이터 구조의 명확성입니다. Django에서 dataclass를 사용하면 다음과 같은 이점이 있을 수 있습니다:

  1. 타입 안정성 : dataclass와 함께 타입 힌트를 사용하면, 코드에 타입에 관한 명시적인 정보를 추가할 수 있습니다. 이로써 코드의 가독성을 높이고, 잠재적인 오류를 줄일 수 있습니다.
  2. 데이터의 불변성 : dataclass에서 frozen=True 옵션을 사용하면 불변 데이터 구조를 만들 수 있습니다. 불변성은 함수형 프로그래밍 패러다임에서 권장되는 특성으로, 예측 가능성과 디버깅 용이성을 높여줍니다.
  3. 명확한 API와 계약 : dataclass를 사용하면, 특정 기능이나 서비스에서 요구하는 데이터의 구조와 형식을 명확하게 정의할 수 있습니다.

이러한 이점을 고려하여 Django와 dataclass의 통합 예제를 다시 검토해보겠습니다.

API 응답 구조화하기 (재검토)

Django REST framework와 같은 API 구축 시, dataclass를 사용하면 응답의 구조를 명확하게 정의하고 해당 구조에 따라 데이터를 직렬화할 수 있습니다.


from dataclasses import asdict
from rest_framework.decorators import api_view
from rest_framework.response import Response

@dataclass
class UserResponse:
    id: int
    name: str
    email: str

@api_view(['GET'])
def user_detail_view(request, user_id):
    user = get_user_from_database(user_id)
    response_data = UserResponse(id=user.id, name=user.name, email=user.email)
    return Response(asdict(response_data))

이 예제에서는 dataclass를 사용하여 API 응답의 구조를 명확하게 정의하였습니다. 이로 인해 API의 반환값의 형태와 내용에 대한 명확한 계약(contract)이 생성되었고, 이를 이용해 직렬화 과정도 간결하게 처리할 수 있습니다.

물론, 모든 상황에서 dataclass를 사용하는 것이 이점만 있지는 않습니다. 적절한 상황과 필요성에 따라 사용하는 것이 중요합니다.

응답의 데이터클래스와 시리얼라이저의 역할 중복

Django REST framework(DRF)에서는 Serializer를 제공하여 복잡한 쿼리셋, 모델 인스턴스를 JSON과 같은 형식으로 쉽게 직렬화할 수 있게 해줍니다. 이러한 기능이 이미 존재하기 때문에 dataclass를 사용하는 것이 중복적이거나 과도한 부분이 있을 수 있습니다.

그러나 dataclass를 API의 응답 구조와 같은 부분에 사용하는 것이 몇 가지 장점이 있을 수 있습니다:

  1. 프레임워크 독립성 : 코드 베이스가 DRF에 너무 의존적이지 않게 하려면, 표준 라이브러리인 dataclass를 사용하여 데이터의 구조를 정의하고 다른 프레임워크나 라이브러리로 전환할 때의 유연성을 높일 수 있습니다.
  2. 타입 체킹 : dataclass와 함께 타입 힌트를 사용하면, 도구나 linters (예: mypy)를 사용하여 코드베이스의 타입 안정성을 체크할 수 있습니다.
  3. 간결한 데이터 변환 : 복잡한 시리얼라이저 로직 없이도 데이터를 빠르게 변환하고, 다른 함수나 메서드로 전달하는 데 유용할 수 있습니다.

예를 들어, DRF의 시리얼라이저를 사용하지 않고, dataclass와 함께 다른 직렬화 방법 (예: dataclasses.asdict(), json.dumps())을 사용하면 다음과 같이 처리할 수 있습니다:


from dataclasses import asdict
from rest_framework.decorators import api_view
from rest_framework.response import Response

@dataclass
class UserResponse:
    id: int
    name: str
    email: str

@api_view(['GET'])
def user_detail_view(request, user_id):
    user = get_user_from_database(user_id)  # 가정
    response_data = UserResponse(id=user.id, name=user.name, email=user.email)
    return Response(asdict(response_data))

그렇지만, 많은 경우에 DRF의 시리얼라이저는 강력하고 효율적입니다. 데이터 유효성 검사, 복잡한 관계 및 중첩된 데이터 구조를 처리하고, 쿼리 최적화와 같은 기능을 제공하기 때문에 많은 Django 프로젝트에서는 DRF의 시리얼라이저를 사용하는 것이 좋습니다.

dataclass의 사용 여부는 프로젝트의 요구사항과 개발팀의 선호에 따라 달라질 수 있습니다.
