python에서 mixin과 상속

kim hyeoncheol

2023년 8월 21일 6 min

Python에서 "mixin"과 "상속"은 서로 관련되어 있지만 동일한 개념은 아닙니다. 그들 사이의 차이점을 이해하기 위해 두 개념을 각각 살펴보겠습니다.

  1. 상속 (Inheritance)

    • 상속은 OOP (객체 지향 프로그래밍)에서의 주요 개념 중 하나입니다.
    • 상속을 사용하면 하나의 클래스가 다른 클래스의 속성과 메서드를 재사용할 수 있습니다.
    • 이를 통해 코드 중복을 줄이고 재사용성을 높일 수 있습니다.
    • 상속받는 클래스를 "자식 클래스" 또는 "하위 클래스"라고 하며, 상속되는 클래스를 "부모 클래스" 또는 "기반 클래스"라고 합니다.

  2. Mixin

    • Mixin은 특정 기능을 클래스에 추가하기 위해 설계된 작은 클래스입니다.
    • Mixin 클래스는 스스로로서는 전체적인 의미가 없으며, 다른 클래스와 함께 사용될 때만 의미를 갖습니다.
    • Mixin은 다중 상속을 통해 여러 클래스에 동일한 기능을 제공할 수 있습니다.
    • Mixin은 클래스의 독립적인 기능을 캡슐화하도록 설계되어 있습니다. 따라서 Mixin을 사용하여 여러 클래스에 걸쳐 해당 기능을 재사용할 수 있습니다.

요약하면, 상속은 한 클래스가 다른 클래스의 속성과 메서드를 재사용하는 메커니즘입니다. 반면 Mixin은 다중 상속을 통해 여러 클래스에 동일한 기능을 추가하는 데 사용되는 작은 클래스입니다. 따라서, 둘 다 클래스 간의 코드 재사용과 관련이 있지만, 사용 목적과 방식에 차이가 있습니다.

코드로 비교해줘

물론이죠. 아래에는 상속과 Mixin을 사용하는 Python의 간단한 예제 코드를 제시하겠습니다.

  1. 상속 (Inheritance)
class Animal:
    def make_sound(self):
        pass

class Dog(Animal):
    def make_sound(self):
        return "Woof!"

class Cat(Animal):
    def make_sound(self):
        return "Meow!"

dog = Dog()
print(dog.make_sound())  # 출력: Woof!

cat = Cat()
print(cat.make_sound())  # 출력: Meow!

위의 코드에서 Dog와 Cat 클래스는 Animal 클래스를 상속받았습니다. 따라서 Dog와 Cat은 Animal의 모든 메서드와 속성을 상속받게 됩니다.

  1. Mixin
class WalkMixin:
    def walk(self):
        return "Walking..."

class SwimMixin:
    def swim(self):
        return "Swimming..."

class Dog(WalkMixin, SwimMixin):
    def make_sound(self):
        return "Woof!"

class Fish(SwimMixin):
    def make_sound(self):
        return "Blub!"

dog = Dog()
print(dog.walk())        # 출력: Walking...
print(dog.swim())        # 출력: Swimming...
print(dog.make_sound())  # 출력: Woof!

fish = Fish()
print(fish.swim())       # 출력: Swimming...
print(fish.make_sound()) # 출력: Blub!

위의 코드에서 WalkMixin과 SwimMixin은 Mixin 클래스입니다. Dog 클래스는 두 Mixin을 모두 상속받아 walk와 swim 메서드를 모두 사용할 수 있습니다. 반면 Fish 클래스는 SwimMixin만 상속받았으므로 swim 메서드만 사용할 수 있습니다.

이처럼 Mixin을 사용하면 특정 기능을 재사용하기 위해 여러 클래스에 동일한 메서드를 제공하는 것이 용이해집니다. Mixin은 주로 다중 상속의 문제점을 피하면서 코드 재사용성을 증가시키기 위해 사용됩니다.

생성자 유무가 가장 큰 차이인가?

Mixin과 일반적인 상속 사이의 주요 차이점은 코드상의 형태보다는 설계 및 사용 목적에 있습니다. 실제로 Python 코드 상에서는 둘 다 클래스 상속의 메커니즘이 사용되기 때문에 두 개념의 차이점이 명확하게 드러나지 않을 수 있습니다. 그럼에도 불구하고 두 가지 접근법의 주요 차이점을 이해하는 것이 중요합니다.

  1. 설계 및 사용 목적:

    • Mixin: Mixin은 일반적으로 작고 독립적인 기능 또는 동작을 재사용하도록 설계된 클래스입니다. Mixin은 스스로 독립적으로 인스턴스화 되어 사용되는 것이 아니라 다른 클래스에 기능을 "믹스"하는 것을 목적으로 합니다.
    • 일반 상속: 일반적인 상속은 "is-a" 관계를 나타내기 위해 사용됩니다. 예를 들면, "Car is a Vehicle"이라는 관계를 표현하기 위해 상속을 사용할 수 있습니다.

  2. 생성자:

    • Mixin: Mixin 클래스는 일반적으로 생성자 (__init__ 메서드)를 포함하지 않습니다. 이는 Mixin이 독립적인 객체로 생성되지 않기 때문입니다.
    • 일반 상속: 하위 클래스는 부모 클래스의 속성과 메서드를 상속받을 수 있으며, 필요한 경우 생성자를 오버라이드하여 추가 속성을 초기화할 수 있습니다.

  3. 사용 방법:

    • Mixin: 클래스가 여러 기능을 포함할 필요가 있을 때 여러 Mixin을 함께 상속받을 수 있습니다. 이는 "다중 상속"의 형태로 이루어집니다.
    • 일반 상속: 일반적으로 하나의 부모 클래스만 상속받아 "is-a" 관계를 형성합니다.

결론적으로, Mixin과 일반적인 상속은 Python에서 형식적으로 비슷한 방식으로 구현될 수 있지만, 둘 사이의 설계 및 사용 목적에는 큰 차이가 있습니다.

javascript에서 mixin과 비슷하게 구현하려면?

JavaScript에는 Python처럼 명시적인 클래스 기반의 Mixin 개념이 없지만, 객체의 프로퍼티와 메서드를 다른 객체와 공유하는 패턴으로 Mixin과 유사한 동작을 구현할 수 있습니다.

JavaScript에서 Mixin 패턴을 사용하는 방법에는 여러가지가 있지만, 여기서는 ES6 클래스와 Object.assign() 메서드를 사용한 간단한 예제를 제시하겠습니다.

JavaScript에서의 Mixin 예제:

// WalkMixin
const WalkMixin = {
    walk() {
        return 'Walking...';
    }
};

// SwimMixin
const SwimMixin = {
    swim() {
        return 'Swimming...';
    }
};

// Dog 클래스 정의
class Dog {
    bark() {
        return 'Woof!';
    }
}

// Mixin 적용
Object.assign(Dog.prototype, WalkMixin, SwimMixin);

// 객체 생성
const myDog = new Dog();

// 메서드 사용
console.log(myDog.bark());   // 출력: Woof!
console.log(myDog.walk());   // 출력: Walking...
console.log(myDog.swim());   // 출력: Swimming...

이 예제에서는 WalkMixin과 SwimMixin이라는 두 개의 Mixin 객체를 정의하였습니다. 그리고 Object.assign() 메서드를 사용하여 Dog 클래스의 프로토타입에 두 Mixin의 메서드와 프로퍼티를 추가하였습니다. 이렇게 하면 Dog 인스턴스는 walk()와 swim() 메서드에 접근할 수 있게 됩니다.

JavaScript에서 Mixin 패턴은 프로토타입 체인을 사용하여 객체 간에 메서드와 프로퍼티를 공유하는 방식으로 구현됩니다.
